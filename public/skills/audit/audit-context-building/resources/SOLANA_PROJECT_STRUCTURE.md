# Solana Project Structure

Understanding the structure of Solana programs for context building.

---

## Anchor Workspace Layout

```
my-solana-project/
├── Anchor.toml              # Workspace configuration
├── Cargo.toml               # Workspace Cargo manifest
├── package.json             # JavaScript/TypeScript dependencies
├── programs/
│   └── my_program/
│       ├── Cargo.toml       # Program-specific dependencies
│       └── src/
│           ├── lib.rs       # Program entry (mod declarations)
│           ├── entrypoint.rs # (Optional) Custom entrypoint
│           ├── instruction/  # Instruction definitions
│           │   ├── mod.rs
│           │   ├── initialize.rs
│           │   └── transfer.rs
│           ├── state/        # Account structs
│           │   ├── mod.rs
│           │   ├── config.rs
│           │   └── vault.rs
│           ├── errors.rs     # Custom error types
│           └── utils.rs      # Helper functions
├── tests/
│   └── my_program.ts        # Anchor test files
├── migrations/              # Deploy scripts
├── target/                  # Build output
│   ├── types/
│   │   └── my_program.ts    # Generated TypeScript types
│   └── idl/
│       └── my_program.json  # Generated IDL
└── app/                     # Frontend (optional)
```

### State/Account Structs

```rust
// state/config.rs
#[account]
pub struct Config {
    pub admin: Pubkey,       // 32 bytes
    pub mint: Pubkey,        // 32 bytes
    pub fee_rate: u64,       // 8 bytes
    pub is_paused: bool,     // 1 byte
    pub bump: u8,            // 1 byte
}
// Space: 8 (discriminator) + 32 + 32 + 8 + 1 + 1 = 82 bytes
```

### Account Validation Structs

```rust
// instructions/transfer.rs
#[derive(Accounts)]
pub struct Transfer<'info> {
    #[account(
        mut,
        has_one = authority,
    )]
    pub vault: Account<'info, Vault>,
    pub authority: Signer<'info>,
    #[account(
        mut,
        constraint = destination.owner == &spl_token::id()
            @ ErrorCode::InvalidTokenAccount,
    )]
    pub destination: Account<'info, TokenAccount>,
    pub token_program: Program<'info, Token>,
}
```

## IDL (Interface Definition Language)

The IDL is auto-generated by Anchor and describes:
- All instruction handlers and their parameters
- All account structs and their fields
- All error codes
- All events

```json
{
  "version": "0.1.0",
  "name": "my_program",
  "instructions": [
    {
      "name": "transfer",
      "accounts": [
        { "name": "vault", "isMut": true, "isSigner": false },
        { "name": "authority", "isMut": false, "isSigner": true }
      ],
      "args": [
        { "name": "amount", "type": "u64" }
      ]
    }
  ],
  "accounts": [
    {
      "name": "Vault",
      "type": {
        "struct": {
          "fields": [
            { "name": "authority", "type": "publicKey" },
            { "name": "balance", "type": "u64" },
            { "name": "bump", "type": "u8" }
          ]
        }
      }
    }
  ]
}
```

## Native Program Structure (Non-Anchor)

```
native-program/
├── Cargo.toml
└── src/
    ├── lib.rs              # Re-exports
    ├── entrypoint.rs       # entrypoint! macro
    ├── instruction.rs      # Instruction enum + packing
    ├── processor.rs        # process_instruction dispatch
    ├── state.rs            # Account data structs
    └── error.rs            # Custom errors
```

### Native Entrypoint

```rust
// entrypoint.rs
use solana_program::entrypoint;

entrypoint!(process_instruction);

// processor.rs
pub fn process_instruction(
    program_id: &Pubkey,
    accounts: &[AccountInfo],
    instruction_data: &[u8],
) -> ProgramResult {
    let instruction = Instruction::unpack(instruction_data)?;

    match instruction {
        Instruction::Initialize { amount } => {
            // Manual account validation required
            let account_info = &accounts[0];
            if account_info.owner != program_id {
                return Err(ProgramError::InvalidAccountOwner);
            }
            if !account_info.is_signer {
                return Err(ProgramError::MissingRequiredSignature);
            }
            // ...
        }
        Instruction::Transfer { amount } => { /* ... */ }
    }
    Ok(())
}
```

## Test Patterns

### Anchor Tests

```typescript
import { Program } from "@coral-xyz/anchor";
import { MyProgram } from "../target/types/my_program";

describe("my_program", () => {
  const provider = anchor.AnchorProvider.env();
  anchor.setProvider(provider);
  const program = anchor.workspace.MyProgram as Program<MyProgram>;

  it("initializes vault", async () => {
    const [vaultPda] = PublicKey.findProgramAddressSync(
      [Buffer.from("vault"), provider.wallet.publicKey.toBuffer()],
      program.programId
    );

    await program.methods
      .initialize()
      .accounts({
        vault: vaultPda,
        authority: provider.wallet.publicKey,
      })
      .rpc();
  });
});
```

## Audit Focus Areas by File

| File | Audit Focus |
|------|-------------|
| `lib.rs` / `#[program]` | Entry points, access control, instruction dispatch |
| `state/*.rs` | Account size, discriminator, PDA seeds, authority fields |
| `instructions/*.rs` | Account validation, constraint logic, CPI calls |
| `errors.rs` | Error coverage, information leakage |
| `utils.rs` | Shared logic, assumption propagation |
| `Anchor.toml` | Program ID, cluster, deploy config |
| `tests/` | Test coverage, edge cases |
